# Wordle Solver — Hint & Auto-Solve Chrome Extension

A Chrome (Manifest V3) extension that solves Wordle-type puzzles using an
information-theoretic algorithm. It reads the live board, tracks the possible
answers, and either drops a single **hint** or runs full **auto-solve**.

It works on **wordleunlimited.org** (primary target), and is built to extend to
NYT Wordle, Quordle (4 boards at once), and other clones via a small adapter layer.

![mode: hint / solver](icons/icon128.png)

## Two modes

- **💡 Hint mode** — at any point while *you* play, it reveals exactly one clue about
  the best word: tap to progressively reveal `# of possible words → starting letter →
  first/last letter pattern → the full word`. It never plays for you.
- **🤖 Solver mode** — shows the statistically best word to play now, with alternates and
  their information gain. `⌨ Type best` types one word for you; `▶ Auto-solve` plays the
  whole game. After every guess it recomputes the next best word.

## The algorithm

The answer is always one of the **2,315** words in the clone's `SOLUTION_LIST`; valid
guesses are the **12,972**-word union with `WORD_LIST`. The solver:

1. **Tracks candidates** — after each guess it keeps only answers consistent with the
   exact green/yellow/gray feedback (duplicate-letter rules handled like real Wordle).
2. **Maximizes information (entropy)** — it scores each possible guess by the Shannon
   entropy of its feedback distribution over the remaining candidates (the 3Blue1Brown
   approach), with a small **answer-bias** so an equally-informative guess that *could*
   win outright is preferred.
3. **Optimized opener** — grid-searched offline over all 12,972 guesses: **`salet`**
   (ties `crate`) minimizes average guesses under this engine.
4. **Exact 2-ply endgame** — once few candidates remain, it re-ranks the top entropy
   guesses by their *true* expected guesses-to-go (depth-limited search with a memoized
   greedy oracle), shaving the long 5/6-guess tail.

### Measured performance (full 2,315-answer set)

| Configuration                       | Avg guesses | Worst | Solve rate |
|-------------------------------------|:-----------:|:-----:|:----------:|
| `raise` opener, greedy entropy      | 3.471       | 6     | 100%       |
| `salet` opener, greedy entropy      | 3.433       | 6     | 100%       |
| **`salet` + exact 2-ply (shipped)** | **~3.43**   | 5–6   | **100%**   |

Reproduce: `node test/treeval.js salet 0.12 100` (fast memoized tree) or
`node test/simulate.js salet` (full simulation). Single in-browser suggestion latency
is < ~150 ms typical, < ~450 ms worst case (the optimized hot path is bit-exact with the
naive reference).

## Install (unpacked)

1. `chrome://extensions` → enable **Developer mode**.
2. **Load unpacked** → select this folder.
3. Open https://wordleunlimited.org/ — the panel appears top-right. Toggle Hint/Solver
   from the panel or the toolbar popup.

## Supported sites

`wordleunlimited.org`, NYT Wordle, Quordle, wordlegame.org, wordleplay.com, plus a
generic `[data-state]` grid fallback. Add a site by extending `adapters.js`.

## Project layout

| File | Purpose |
|------|---------|
| `manifest.json` | MV3 manifest, content-script matches |
| `words.js` | Embedded answer + allowed lists (generated by `tools/extract_words.py`) |
| `solver.js` | Pattern logic, entropy, answer-bias, 2-ply endgame, board state machine |
| `adapters.js` | Per-site board reading (shadow DOM aware) + reliable typing |
| `content.js` | Overlay UI, mode logic, multi-board orchestration, auto-solve loop |
| `popup.html` / `popup.js` | Toolbar popup (mode toggle, show/hide) |
| `test/` | `simulate.js` (full sim), `treeval.js` (fast memoized evaluation) |

The word lists are derived from the clone's own publicly-served `main.js`, so the solver
only ever guesses words the game accepts.
